# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: [your-email@example.com]

You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include:

- Type of issue (e.g., buffer overflow, data exposure, denial of service)
- Full paths of source file(s) related to the issue
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## What to Expect

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 1 week
- **Fix timeline:** Depends on severity (critical: ASAP, high: 1-2 weeks, medium/low: next release)

## Scope

Security issues in:
- Data integrity (data loss, corruption)
- Authentication/authorization (if applicable)
- Denial of service
- Information disclosure
- Dependency vulnerabilities

## Recognition

We appreciate responsible disclosure and will acknowledge security researchers in release notes (unless you prefer to remain anonymous).
